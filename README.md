# Distributed Event-Driven Notification System

A slow-burn backend engineering lab built in Go, where a notification system becomes the excuse to study the real machinery of computer systems: requests turning into work, work turning into events, events crossing boundaries, failures forcing retries, queues absorbing pressure, workers competing with time, state hardening inside storage, and architecture emerging not from buzzwords but from the stubborn realities of concurrency, scale, delivery, and control.
